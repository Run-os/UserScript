## 操作坑（Windows + 中文路径 git）
- `git mv` 在「已暂存的中文重命名 + Windows」下易报 `No such file or directory`，或残留同名未跟踪副本（Unicode 规范化 NFC/NFD 不一致）。稳妥流程：`git reset -q`（不动工作区）→ bash `mv` 移动 → `git add -u`（保留 R 历史、避免旧结构文件被「复活」回仓库）。
- 用 `git show HEAD:旧路径` 可从对象库恢复误删 / 丢失文件（即使工作区已无）。
- `git add -u` 会顺带暂存所有「本地已删未提交」的旧文件（D）；若只想暂存本次改动，需事后 `git restore --staged <path>` 取消。

## 脚本验证 workflow（browser-use 全自动）【用户明确约定】
- **方式**：调试/改完脚本后，用 `browser-use` 在浏览器里**全自动验证**（非人工目测）。
- **必须新开网页**：因 userscript 由油猴/ScriptCat 管理器在**页面加载时注入**，脚本代码**无法写入已打开的网页**（事后 JS `eval` 进不去管理器的隔离 world，且语法校验只是静态、不等于真能跑）。故验证必须**新开一个网页（fresh navigation）**，让管理器在加载瞬间自然注入脚本。
  - 实操：`browser-use` 连上用户已开远程调试的 Chrome → `new_tab(<目标URL>)` 打开新标签 → 等待加载 / 管理器注入 → 用 `js()`/`page_info()`/AX 树断言行为 → 必要时 `click_at_xy(x,y)` 发**真实 CDP 鼠标事件**（带用户激活，可解锁静音/播放策略）。
- **URL 规则**：新开网页的网址**若用户未指定，必须先询问用户**（不擅自猜测目标页，尤其百度网盘需登录态 + 具体视频 path）。
- **前置（用户配合）**：①Chrome 开启 `chrome://inspect/#remote-debugging` 的"Allow remote debugging"；②ScriptCat/油猴里**「从文件更新」目标脚本**（管理器不自动读磁盘改动，否则验证的是旧版，见下"运维坑"）；③刷新页面让新脚本生效。
- **旧结论已作废**：本站历史日志 §95 曾记"`browser-use new_tab` 创建的标签扩展常不注入、务必在用户手动打开的标签验证"——**该结论错误，现已丢弃**（用户于 2026-07-22 22:56 明确指示忽略）。验证统一用 `new_tab(<目标URL>)` 新开网页，管理器随页面加载自然注入，**无需也不应**退回"在用户手动打开的标签上验证"。`Target.attachToTarget` 挂标签技巧仅作特殊排查（如查看某已存在标签）的辅助手段，不作为验证主路径。
- **运维坑（复用，勿忘）**：ScriptCat 不会自动同步磁盘编辑；每次改完 `.user.js` 必须让用户在 ScriptCat「从文件重新导入/更新」+ 手动刷新页面，否则验证的是旧版。调试日志建议用 `window.__BD_XXX_LOG__ = window.__BD_XXX_LOG__ || []` 累积、再由 `Runtime.evaluate('JSON.stringify(...)')` 读取，避免依赖已流逝的 console。
