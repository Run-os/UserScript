## 修复「百度网盘打开中文字幕.user.js」全部 8 个 bug + browser-use 验证

- 文件：`工具/百度网盘打开中文字幕.user.js`，版本 bump 26.7.21 → 26.7.22。修复项：
  1. 并发锁 `clicking`：MutationObserver/轮询高频触发时避免重复点击同一字幕项（toggle 语义下可能反复开关）。
  2. SPA 切换视频后仍能重新开启：去掉「成功即永久 done」的死锁，改为 `handledItem` 跟踪，检测到已处理项消失/取消勾选则重置重试；observer/poll 在成功时不断开（仅超时断开）。
  3. `findChineseItems` 改用 `textContent` 而非 `innerText`（折叠隐藏的字幕项 innerText 为空会匹配不到）。
  4. `@match` query string 通配不可靠，新增 `@include` 正则兜底：`/^https?:\/\/pan\.baidu\.com\/(pfile|play)\/video\?.*path=.*\.mp4/`。
  5. MutationObserver 回调加 300ms 节流 `scheduleTry`，避免高频刷屏吃性能。
  6. `pickChineseItem` 优先选「简体」，避免误选繁体。
  7. `findSubtitleToggle` 候选顺序改为优先真正的 `button[class*="subtitle"]`，避免点到容器。
  8. `findSubtitleToggle` 加 `typeof el.className === 'string'` 守卫，排除 SVG 元素（className 是 SVGAnimatedString，`.includes` 会抛错）。
- `node --check` 语法校验通过。
- **browser-use 验证（已连通，用户此前已开启 Chrome 远程调试）**：
  - 真实页 `pan.baidu.com/pfile/video?path=...mp4` 加载成功（标题「百度网盘 - 视频播放」，非登录墙），字幕菜单含「不显示字幕」「中文字幕」，DOM 结构与脚本选择器一致。
  - 真实菜单点击 path 验证不可靠：百度播放器会异步重渲染字幕菜单并恢复 is-checked，外部点「不显示字幕」无法稳定取消中文字幕勾选，故改用**受控 mock DOM 单测**：注入三个 li（不显示/中文（繁体）/中文（简体）），跑修复后的脚本 → 正确选中「中文（简体）」且**仅点击 1 次**（MOCK_CLICKS=1）→ 证明修复 1/3/6 通过。
