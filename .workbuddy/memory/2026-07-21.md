## browser-use 安装与技能注册

- 用 `uv tool install --python 3.12 browser-use` 安装成功，版本 0.1.6（当前最新），Python 3.12.13。可执行文件在 `~/.local/bin`（已在 PATH），提供命令：browser-use / browser / bu / browseruse / browser-use-tui。
- **重要坑（Windows）**：`browser-use skill install` 在 Windows 上必然失败。它内部会执行 `uv tool install --python 3.12 --upgrade --force browser-use` 对自身强制重装，而运行中的 `.exe` 锁住了 `AppData\Roaming\uv\tools\browser-use\Scripts` 目录，导致 `os error 5 拒绝访问`；且会先删掉 `browser_use` 模块再失败，留下损坏安装（`ModuleNotFoundError: No module named 'browser_use'`）。修复方法：`uv tool uninstall browser-use && uv tool install --python 3.12 --upgrade --force browser-use`。
- 替代注册技能方式（来自 browser-harness install.md 回退方案）：`browser-use skill > SKILL.md`。已写入 `C:\Users\liuyz\.workbuddy\skills\browser-use\SKILL.md`（含 name/description frontmatter，可作 WorkBuddy 技能）。
- **浏览器连接未完成**：`browser-use --doctor` 显示 chrome running=ok，但 daemon alive=FAIL、active connections=0。原因是 Chrome 远程调试权限未开启。需用户手动：打开 `chrome://inspect/#remote-debugging` → 勾选 "Allow remote debugging for this browser instance" → 弹窗点 Allow → 再跑 `browser-use <<'PY' \n print(page_info()) \n PY` 验证。此步骤必须用户操作。
