## 百度网盘打开中文字幕.user.js — 新增「启动后自动播放」功能（v26.7.22 → 26.7.23）
- 需求：脚本启动后自动播放视频。
- 实现（复用现有 MutationObserver + 轮询机制）：
  - `tryPlay()`：先 `video.play()` 有声播放；被浏览器策略拦截（NotAllowedError）时退化为 `video.muted=true` 再 `play()`（静音播放浏览器始终允许，保证视频一定动起来）。

  - **静音自动播放始终允许**；有声自动播放需用户激活（手势）或高 Media Engagement Index。

  - 注入脚本 → 视频从暂停自动开始播放（静音兜底，currentTime 推进）→ PASS。
  - 用 `click_at_xy(x,y)`（真实 CDP 鼠标事件，带用户激活）点击视频区外中性点 → `muted=false` 且 `paused=false`、currentTime 继续推进 → PASS（声音恢复且继续播放）。

## ⚠️ 关键运维发现：ScriptCat 不自动读磁盘改动 + browser-use 指定标签技巧
- **ScriptCat 不会自动同步磁盘编辑**：实网标签 `window.__bdArtPlayerReplaced===false`、`window.Hls==='undefined'`、`NO_GM_INFO` → 装的仍是旧版，我磁盘改动没进浏览器。故每次改完 .user.js，**必须让用户在 ScriptCat 里「更新/从文件重新导入」该脚本，再手动刷新页面**，否则验证的是旧版。
- **browser-use 指定标签调试法**：`page_info()`/`js()` 只作用于 daemon 默认标签。要调试某个已存在标签：①`cdp("Target.getTargets")` 取**完整**（勿截断）`targetId` ②`cdp("Target.attachToTarget", targetId=FULL, flatten=True)["sessionId"]` ③`cdp("Runtime.evaluate", expression=JS, session_id=sid, returnByValue=True)` 读 DOM/全局。②`browser-use new_tab(url)` 创建的标签扩展常不注入（勿用其验证 userscript），务必在**用户手动打开**的标签上验证。
- ⚠️ 调试日志写法：脚本内用 `window.__BD_ART_LOG__ = window.__BD_ART_LOG__ || []` 累积日志，再由浏览器 `Runtime.evaluate('JSON.stringify(window.__BD_ART_LOG__)')` 读取，避免依赖已流逝的 console。

## 拆解付费脚本的私有依赖库（artPlugins.js / joysound.js）
- 用户要区分"哪些涉及付费、哪些可借鉴"。已 `curl` 下载两库源码（artPlugins.js 34KB / joysound.js 91KB，均单行压缩）逐段分析。
- **artPlugins.js（v2.1.0）内含付费门控系统**：
  - 头部 `window.cloudbase.init({env:"js-3gyg0gq54003de57", accessKey:"eyJhbGciOiJSUzI1Ni...JWT"})` —— 作者自己的腾讯云开发环境 + JWT 访问密钥。
  - `h.callFunction({name:"pan", data:{user,host,uid,gminfo,uinfo}})` → 返回 `{updatedAt, expire_time, ec, em, ...}`（会员过期时间）。
  - 校验函数 `m()`：判断 `uid 匹配 + updatedAt + 1天缓存(864e5) + key 逆向 btoa 校验`，决定是否放行高级功能。
  - 右键菜单"赞赏作者" → "开通功能"(`g()` 开支付页) / "鼓励一下"。即**会员付费墙在库内**，高级功能需 `expire_time` 有效。
  - **但核心播放代码是开放的**：quality 从 `file.resolution` 正则 `width:(\d+),height:(\d+)` 提取（与我的 v2 一致）；`subtitle:{url,type:"auto",style:{color:"#fe9200",bottom:"5%",fontSize:"25px",fontWeight:400}}`；`ASPECT_RATIO:["default","自动","4:3","16:9"]`；`t.on("video:ended",()=>{if(auto-next&&playlist){...}})` 连播；`filter`=CSS `saturate/brightness/contrast`；`screenshot:!0`；`autoplay/autoPlayback` + `recoverMediaError` 容错。
- **joysound.js（v1.0.3）=「音量增强/音质增强」**，纯客户端 Web Audio API + AudioWorklet + WASM（base64 内嵌 wasm DSP 模型）：
  - `window.AudioContext||webkitAudioContext` → `createMediaElementSource(视频元素)` → `createGain` → `connect(destination)`，靠 `_worklet`/`_jsww` + WASM 做音频增强；`setEnabled/setVolume` 控制。
  - **库本身无付费门控**（无 cloudbase/key 校验），但 artPlugins 仅在会员功能标志下才显示"音量增强"开关 → 实践中属付费功能。
  - 概念可借鉴：把 `<video>` 经 `AudioContext` 路由即可做增强；但**完整 WASM DSP 不必复刻**——用公开 `createGain` 提音量 / `createBiquadFilter` 做 EQ 就能达到"音质增强"效果，零私有依赖。
- **结论（用户脚本可借鉴 / 不可借鉴）**：
  - ✅ 可白嫖复刻（纯 ArtPlayer/Web Audio 公开能力，与会员无关）：字幕样式(color/bottom/fontSize)、画面比例 4:3/16:9、连播下一集、CSS 滤镜色彩调整(saturate/brightness/contrast)、截图、自动播放+错误恢复(recoverMediaError)、倍速菜单。
  - ❌ 不必/不可借鉴（付费墙或私有）：cloudbase 会员校验 + `callFunction pan` + `expire_time` 整套付费系统；joysound 内嵌 WASM 音频模型（用公开 Gain/EQ 替代即可）。
  - 我方脚本定位：只复刻 ✅ 部分，彻底不碰 ❌ 部分，保持零授权、源码透明。
