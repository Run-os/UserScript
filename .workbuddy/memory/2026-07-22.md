## 百度网盘打开中文字幕.user.js — 新增「启动后自动播放」功能（v26.7.22 → 26.7.23）
- 需求：脚本启动后自动播放视频。
- 实现（复用现有 MutationObserver + 轮询机制）：
  - `tryPlay()`：先 `video.play()` 有声播放；被浏览器策略拦截（NotAllowedError）时退化为 `video.muted=true` 再 `play()`（静音播放浏览器始终允许，保证视频一定动起来）。
  - `armUnmuteOnInteraction()`：静音兜底后，在 document 上挂一次性 capture 监听（click/keydown/touchstart），用户首次真实交互时 `muted=false`；并在回调内（真实用户手势中）若 `paused` 则 `play()` 兜底恢复播放。
- 关键认知（浏览器自动播放策略）：
  - **静音自动播放始终允许**；有声自动播放需用户激活（手势）或高 Media Engagement Index。
  - **合成事件（dispatchEvent / .click()）不提供用户激活**：静音播放中用代码 `muted=false` 会被浏览器重新评估并暂停视频——这是测试假象，真实点击不会。
  - 播放器是 Video.js：大播放按钮 `button.vjs-big-play-button`（title「播放视频」）；**合成点击播放按钮无法启动播放**（不算手势），故未采用点击按钮方案。
- browser-use 验证（真实页面，用户已开启 Chrome 远程调试）：
  - 注入脚本 → 视频从暂停自动开始播放（静音兜底，currentTime 推进）→ PASS。
  - 用 `click_at_xy(x,y)`（真实 CDP 鼠标事件，带用户激活）点击视频区外中性点 → `muted=false` 且 `paused=false`、currentTime 继续推进 → PASS（声音恢复且继续播放）。
  - 教训：验证"解除静音"必须用 `click_at_xy` 真实点击，`js()` 里 dispatchEvent 因缺用户激活会误报暂停。
- 语法 `node --check` 通过。未 commit（沿用惯例，提交由用户决定）。

## 百度网盘打开中文字幕.user.js — 改为「绝不静音、带声音自动播放」（v26.7.25 → 26.7.26）
- 需求：用户反馈"有时视频有声音但画面不动"，并明确要求**不要静音起播**。
- 调研结论（Chrome 自动播放策略，硬性、脚本无法绕过）：
  - 静音自动播放**永远允许**；有声自动播放仅当：①用户与该域名交互过 ②该站点 MEI（媒体互动指数）足够高（常看有声视频即达标）③装成 PWA ④开发者开关 `--autoplay-policy=no-user-gesture-required`。
  - 因此脚本在"被拦截"时只有两条路：A 静音起播（旧方案，用户不要）B 挂监听等用户首次真实交互后带声音播（本次采用）。
  - "有声音但画面不动"≈百度 Video.js 渲染坑：有声自动播放被放行后大播放按钮/海报层未清掉，盖住画面 → 用代码隐藏遮罩修。
- 实现：
  - 新增 `CONFIG.mutedFallback=false`（默认 False=绝不静音；翻 True 可回退旧静音行为）。
  - `tryPlay()` 有声被拦截时：不再 `video.muted=true`，改为 `armSoundPlayOnInteraction(video)`——挂一次性 capture 监听（click/keydown/touchstart），用户首次真实交互时 `video.play()` 带声音播；并弹 `toastr.info('▶️ 点击页面任意位置即可带声音播放')` 一次提示。
  - 新增 `ensureVideoVisible()`：播放成功后给 `.video-js` 容器加 `vjs-has-started vjs-playing`、移除 `vjs-paused`、隐藏 `.vjs-big-play-button`，解决"有声音但画面不动"。
  - 旧 `armUnmuteOnInteraction` 仅保留给 `mutedFallback=true` 分支用。
- browser-use 验证（真实页面，用户已开远程调试）：
  - 拦截场景（覆写 `HTMLMediaElement.prototype.play` 恒 reject 模拟浏览器拦截）：注入后 `muted=False`（不再静音）✓；提示气泡出现 ✓。
  - 恢复真实 play 后用 `click_at_xy(x,y)` 真实点击视频区外中性点：`muted=False, paused=False, currentTime=1.47, .vjs-big-play-button display=none` → **带声音播放 + 遮罩隐藏 PASS**。
  - ⚠️ 测试沙箱坑：toastr 依赖 jQuery，若**异步同时加载**两者、toastr 先于 jQuery 就绪，则其内部 `$.extend` 引用为 undefined → `toastr.info` 抛 `TypeError: Cannot read properties of undefined (reading 'extend')`、气泡为空。这是 harness 时序假象；**真实 Tampermonkey `@require` 按顺序同步加载**（jQuery 在 toastr 之前）不受影响。验证 toastr 脚本时须先 `await window.jQuery` 再加载 toastr。
- 语法 `node --check` 通过。未 commit。
